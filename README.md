# Snapture — Stream Deck plugin

Trigger [Snapture](https://github.com/Este2013/Snapture) captures from a Stream Deck.

Requires **Snapture ≥ 1.1.0** running (the plugin talks to its local control
server on `127.0.0.1:48910`) and **Stream Deck 6.5+**. Windows only.

## Actions

- **Snapshot** — take a still image.
- **Snapture** — record the screen. While recording, every Snapture key turns
  red with a pulsing dot and shows the elapsed time (your key title is restored
  when recording stops).
- **Open last snap** — open the most recent capture; choose *Any snap*, *Only
  snapshots*, or *Only snaptures*.

Snapshot and Snapture each offer, per key:

| When pressed | Behaviour |
| --- | --- |
| Open the picker | Opens Snapture's selection overlay; pick a starting mode (Default / Display / Window / Custom). |
| Use a specific display | Captures the chosen display directly. |
| Use a specific window | Captures the chosen window (falls back to matching its title if the handle is stale). |
| Repeat the last capture | Re-runs this session's last capture of that kind. |
| Open the library folder | Opens the video/snapshot library. |

Plus a **Format** choice (defaults to Snapture's app default). Windows/displays
are read live from the app. If Snapture is missing or too old, the key shows an
alert and the reason is logged instead of failing silently.

## Build

```bash
npm install
npm run build        # bundles src/ -> com.este.snapture.sdPlugin/bin/plugin.js
```

Icons are generated by `./generate-icons.ps1` (already committed under
`com.este.snapture.sdPlugin/imgs/`).

## Install / develop

```bash
streamdeck link com.este.snapture.sdPlugin   # symlink into Stream Deck
streamdeck restart com.este.snapture         # (re)start after a build
streamdeck pack com.este.snapture.sdPlugin   # package a distributable
```

## How it connects

The plugin (Node) opens a TCP connection to Snapture's control server and speaks
its newline-delimited JSON protocol (see `docs/StreamDeck.md` in the Snapture
repo). It reconnects automatically and reports the app version for feature gating.
